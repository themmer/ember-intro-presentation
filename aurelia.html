  • Convention over Configuration, but can override
  • Code changes - auto reloads
  • Testability, maintainability, extensibility, learnability
  • Custom CLI generators
  • Modular code packages => portable other places
  • Unit and Integration testing
  • Commercial Support
  • Aurelia's dependencies are only polyfill

Components (Elements)

  • au generate element component-name
     
  • Creates .js and .html in src/resources/elements folder

Element Template

          <template>

            <input value.bind="firstName">

            <input value.bind="lastName">

            <button click.trigger="sayHello()">Say Hello</button>

          </template>

Element Javascript

          export class Hello {

            firstName = 'John';

            lastName = 'Doe';

            sayHello() {
              alert(`Hello ${this.firstName} ${this.lastName}`);
            }

          }

Require

  • Imports resources (js/css) for templates
     
            <require from="bootstrap/css/bootstrap.css"></require>
            <require from="./currency-format"></require>

Bindings

  • one-time (one-way)
  • one-way
  • two-way
  • bind (auto select - two-way for forms)
     
  • I.E. Binds attribute of value for input whether data or form changes
E.G. <input type="text" value.two-way="lastName">
