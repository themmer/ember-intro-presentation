        ** Why High-Fidelity (HF)?
          Realistic for users/customers/stakeholders
          Forces you to think
          Find problems earlier than real implementation
          Keep focus on user experience up-front

        ** Advantages of HF
          Page Hierarchy
          User Workflows
          Complex Components
          Interactive

        User / Stakeholder Benefits
          Person won't get hung up on incorrect or repeated data
          Less `squirrels` of what about this
          	'That's not the correct format of that data!'
          	'The data is repeating!'
          	'This button doesn't work!'
          See Responsive design and show data layout issues
          Some people need a visual to approve or find problems
          	If the stakeholder doesn't know this could guide them
        When stakeholders don't know what they want
          I've seen 3 iterations on projects before (server work included)
            Introduces bugs, frustrates most workers
        Can still be used with low-fidelity prototyping to start

        ** Disadvantages of HF with JS
          More expensive than low-fidelity if prototype is thrown away
          Requires specialized skills of UX design and/or programming
          Requires experience on the programming and design side
            Along with the frameworks and technologies chosen

          Takes longer than Low-Fidelity
            More expensive and/or time
            Entrepreneurship requires quick turnaround

          Simple tasks that add up
          	Troubleshooting

          Spending time on server work

        ** Advantages of using Ember.js for HF Prototyping
          Automation of creation of files (CLI)
          Lots of available addons
            Creating a new project is easy
          Local data setup that can be switched to a server (ember-cli-mirage)
            `faker` creates random data and specific data rules (e.g. Length)
            Fake data can be used for testing too
          Auto Reload Browser with changes
          Out-of-the-box technologies already incorporated and ready to start
          Each project gets easier due to similarities and reuse

        ** Sx5 Rapid Prototyping for UX Design
          Strategy
          	What is the big goal of the product?
        	What does the user want?
        	What does the business want?

          Scope
        	Creating user stories (actions)
        	List objectives for the software
        	How does the product help the users?
        	How many clicks does it take to get to each essential function?
        	What type of user is interested in this application?

          Structure
        	How do all the pieces of the site fit together and behave?
        	How does the user find what they are looking for in an intuitive way?
        	What are the flow of actions and interactions throughout the site?
